JPEG Metadata Filter

Strips EXIF, GPS, camera data, and comments from JPEG images crossing a security boundary. Only pixel data passes through. Operates inline between Linux VMs at wire speed using the same VM diode architecture.

Security Properties

  • EXIF removal — All APP1 segments (camera make/model, lens, settings, timestamps) stripped
  • GPS data removal — Geolocation coordinates embedded in EXIF are completely removed
  • Comment removal — COM segments that may contain sensitive text or annotations stripped
  • Pixel-only output — Only the structural segments needed to decode the image pass through
Current Implementation

seL4 Microkit + C

The system uses the same VM diode architecture as the Linux VM Data Diode, but the trusted intermediary parses JPEG data instead of forwarding raw Ethernet frames. The filter walks the JPEG segment structure and only emits segments that contain pixel data.

Key Code Highlight jpeg_filter.c 
/* JPEG segment markers */
#define JPEG_SOI  0xFFD8   /* Start of Image     */
#define JPEG_EOI  0xFFD9   /* End of Image       */
#define JPEG_SOS  0xFFDA   /* Start of Scan      */
#define JPEG_APP1 0xFFE1   /* EXIF / GPS data    */
#define JPEG_COM  0xFFFE   /* Comment            */

static int is_safe_marker(uint16_t marker) {
    /* Allow only structural segments needed for decoding */
    switch (marker) {
        case 0xFFD8: /* SOI */
        case 0xFFC0: /* SOF0 */
        case 0xFFC4: /* DHT  */
        case 0xFFDB: /* DQT  */
        case 0xFFDA: /* SOS  */
        case 0xFFD9: /* EOI  */
            return 1;
        default:
            return 0;  /* Strip everything else */
    }
}

size_t filter_jpeg(const uint8_t *in, size_t in_len,
                   uint8_t *out, size_t out_cap) {
    size_t r = 0, w = 0;
    while (r < in_len - 1) {
        uint16_t marker = (in[r] << 8) | in[r+1];
        if (is_safe_marker(marker)) {
            /* Copy safe segment to output */
            size_t seg_len = get_segment_length(in, r, in_len);
            memcpy(out + w, in + r, seg_len);
            w += seg_len; r += seg_len;
        } else {
            /* Skip unsafe segment (EXIF, GPS, COM, etc.) */
            r += get_segment_length(in, r, in_len);
        }
    }
    return w;
}

Fiducia Specification

The Fiducia version of this system will use the same one-way channel topology as the data diode. The JPEG parsing and filtering logic stays entirely within the C/Pancake implementation of the jpeg_filter protection domain — Fiducia verifies the structural information flow guarantee.

  • Same topology as data diode — The .fis system declaration uses the identical one-way channel pattern
  • JPEG parsing stays in C/Pancake — The segment-level filter logic remains in the imported C implementation
  • Structural flow guarantee — Fiducia verifies at compile time that no image data or metadata can flow from LOW back to HIGH
